# ML_assignment4

## კონკურსის მიმოხილვა

Facial Expression Recognition (FER-2013) კონკურსის მიზანია ადამიანის სახის ემოციების ამოცნობა და კლასიფიკაცია 7 სხვადასხვა კატეგორიაში (Angry, Disgust, Fear, Happy, Sad, Surprise, Neutral). ამოცანა სრულდება 48x48 პიქსელის ზომის, შავ-თეთრ გამოსახულებებზე. ეს არის კლასიკური Computer Vision და Deep Learning-ის პრობლემა.


## მიდგომა

ეს ამოცანა წარმოადგენს მრავალკლასიანი კლასიფიკაციის პრობლემას. გამოვიყენე სხვადასხვა სტრუქტურის Convolutional Neural Network- ები (CNN) და ავირჩიე იტერაციული მიდგომა. დავიწყე უმარტივესი Baseline მოდელით,. შემდეგ ეტაპზე გავაკეთე უფრო ღრმა მოდელი (Deep CNN), რომელმაც აჩვენა ძლიერი Overfitting-ი. მესამე ეტაპზე გამოვიყენე რეგულარიზაციის ტექნიკები (BatchNorm და Dropout), რამაც ცოტათი გააუმჯობესა მოდელის განზოგადების უნარი და შედეგებიც. საბოლოოდ, მეოთხე იტერაციაში გამოვიყენე VGG-ს სტილის 8-შრიანი არქიტექტურა, Data Augmentation და Global Average Pooling-ით, რამაც მოდელის სიზუსტე, სხვებთან შედარებით, ყველაზე მაღალზე აიყვანა.

ყველა ექსპერიმენტი, ჰიპერპარამეტრების Tuning და მეტრიკები დალოგილია **Weights & Biases (WandB)**-ის გამოყენებით.

## მოდელების იტერაციები და ექსპერიმენტები

ყველა მოდელისთვის გამოვიყენე ჰიპერპარამეტრების Grid Search:
- **Batch Sizes:** `[64, 32]`
- **Learning Rates:** `[0.001, 0.0005, 0.0003]`

### 1. Small CNN (Baseline)
თავდაპირველად გავტესტე უმარტივესი არქიტექტურა, რომელსაც ჰქონდა მხოლოდ 2 Convolutional შრე. მოდელმა სწრაფად ისწავლა, მაგრამ ვერ შეძლო კომპლექსური მახასიათებლების აღქმა. სიზუსტე იყო დაბალი (**52.86%**), რამაც დაადასტურა, რომ უფრო ღრმა ქსელი მჭირდებოდა.

### 2. Deep CNN 
მეორე ეტაპზე ავაშენე 4 Convolutional შრიანი მოდელი ერთი უზარმაზარი Fully Connected შრით (18.8 მილიონი პარამეტრი).
- **შედეგი:** მოდელმა აჩვენა კატასტროფული Overfitting. Train Accuracy ავარდა თითქმის **99.17%**-მდე, ხოლო Validation Accuracy გაიჭედა **58.79%**-ზე. დაახლოებით 6% ით გაუმჯობესდა წინა მოდელის შედეგი, თუმცა გაჩნდა უზარმაზარი **~40.69%-იანი Gap-ი** train-სა და ვალიდაციის სეტს შორის. მოდელმა უბრალოდ დაიზეპირა ტრენინგ დატასეტი.

### 3. Deep BN & Dropout CNN 
წინა მოდელის პრობლემის გადასაჭრელად, ზუსტად იგივე არქიტექტურას დავუმატე **Batch Normalization** და **Dropout** (0.25 კონვოლუციურ შრეებში და 0.5 Fully Connected შრეში).
- **შედეგი:** რეგულარიზაციამ რაღაც მხრივ იმუშავა, ვინაიდან ნეირონები რანდომულად ითიშებოდნენ. gap -შემცირდა და ასევე წინებთან შედარებით validation accuracy-ც გაიზარდა 62%-მდე.

### 4. Deepest AUG CNN 
საბოლოო მოდელისთვის გამოვიყენე VGG-ის მსგავსი არქიტექტურა **8 Convolutional შრით**. ვინაიდან ასეთი ღრმა მოდელი კვლავ დაიწყებდა დაზეპირებას, დავამატე ორი კრიტიკული ტექნიკა:
1. **Data Augmentation:** `transforms.RandomHorizontalFlip()` და `transforms.RandomRotation(10)`. ეს უზრუნველყოფს იმას, რომ ქსელი ყოველ epoch-ზე ხედავს ოდნავ შეცვლილ სურათს.
2. **Global Average Pooling (GAP):** `AdaptiveAvgPool2d(1)`-ის გამოყენებით Fully Connected შრის პარამეტრები 18 მილიონიდან ჩამოვიდა სულ რაღაც 130,000-მდე.
- **შედეგი:** მოდელმა აჩვენა ნორმალური სიზუსტე **67.46%**. gap- უფრო დაბლა ჩამოვიდა წინებთან შედარებით.
---

| მოდელი | საუკეთესო Validation Accuracy | Train Accuracy | Gap (train_accuracy - validation_accuracy) |
|--------|-----------------------------|----------------|-------------------|
| **Small CNN** | 52.86% | 67.39% | 16.01% |
| **Deep CNN** | 58.79% | 99.17% | 40.69% |
| **Deep_BN_DR_CNN** | 62.55% | 96.47% | 34.67% |
| **Deepest_BN_DR_AUG_CNN** | **67.46%** | 93.15% | **27.20%** |


## Wandb tracking

თითოეული ექსპერიმენტის მიმდინარეობისას ყველა epoch-ze, WandB-ში რეალურ დროში ილოგებოდა შემდეგი მეტრიკები:

- train_acc და val_acc 
- train_loss და val_loss 
- acc_gap (სხვაობა Train-სა და Validation სიზუსტეებს შორის)
  
თითოეული ნოუთბუქის (არქიტექტურის) ტრენინგის დასრულების შემდეგ, ავტომატურად ეშვება საბოლოო Evaluation სკრიპტი. ეს სკრიპტი ირჩევს და ტვირთავს კონკრეტულად ამ მოდელის ყველაზე მაღალი Validation Accuracy-ის მქონე ეპოქის წონებს (`best_model.pth`) და WandB-ში აგზავნის შემდეგ მეტრიკებს:

- **Confusion Matrix:** იმის სანახავად, თუ რომელ კლასებში უჭირს მოდელს (მაგ: Sad vs Neutral).
- **Precision, Sensitivity, Specificity, Accuracy და F1-Score**.

- ყველა run, გრაფიკები და დატრენინგებული მოდელის არტეფაქტები (`.pth` ფაილები) შენახულია WandB-ზე.
